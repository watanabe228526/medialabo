本テーマについて

このテーマでは,HTMLでWeb ページの構造を, CSSで見た目の装飾を, JavaScriptで動きのあるページを実現する方法を学び、これらを組み合わせて Web ページを構成する方法を学びます。

プロジェクト課題について

このプロジェクト課題では、世界の天気が調べることができます。カイロやモスクワ、ヨハネスブルクなど12地域から選択し、その地域に設定された6桁から7桁のIDをテキストボックスに入力し検索ボタンを押すと最高気温と最低気温が出てきます。またこの天気予報のページでは、天気をイメージした空色を多く使いました。最高気温と最低気温の2つしか出すことができないので晴れや風向きなど出すことができればなおよかったと思います。

数当てゲーム

この数当てゲームでは、乱数で 1〜10の値を1つ決めて、「答え」を設定し、ユーザは答えを予想して、数を最大で3回入力できます。ユーザの入力が外れた場合は、入力よりも答えが大きいか小さいか表示して、3回以内に答えの数を当てればユーザの勝ち、そうでなければ負けというゲームです。ページには、テキストボックスとボタンが設置されていて、ユーザはテキストボックスに予想した数を入力して、 ボタンを押すと判定します。

各授業回の学習事項

第1回WWWとHTML

第1回の授業では、ウェブページの構造とコンテンツを作成する方法について学びました。タグ、属性、および要素の使用方法を理解し、段落、見出し、リンクなどの基本的な構成要素を組み合わせてページを構築しました。また、フォームや画像の埋め込みなど、さまざまな要素を使用してインタラクティブなコンテンツを作成する方法も学びました。

演習1-4: HTMLのさまざまな要素

第2回CSSによるWebページの装飾

第2回の授業通して、ウェブページのスタイルやレイアウトをデザインする方法について学びました。セレクタ、プロパティ、値の組み合わせを使って、テキストの装飾やボックスモデルの調整などの演習を行いました。

演習2-3:ウェブブラウザ説明ページの装飾

第3回JavaScriptの基本

第3回の授業では、JavaScriptの基本文法やjavaとJavaScriptの違い、デバック学びました。Javaはコンパイルされる静的型付け言語で、JavaScriptは動的型付けのインタプリタ言語です。

演習3-6:オブジェクトの参照

第4回DOMとJavaScript

第4回の授業では、JavaScriptの基本とDOM操作を学びました。DOMはツリー構造を持ち、各要素（タグ）、属性、テキストなどがノードとして表されます。DOM と JavaScript を組み合わせることによって，動きのある Web ページを実現できるようになります

演習4-2から4-5:演習の成果

第5回イベント処理

第5回の授業では、イベント処理について学びました。イベント処理とは、イベントが発生したときにJavaScriptプログラムが動作するような処理のことです。色々なイベント処理がありその中でもわかりやすいイベント処理を用いて実験しました。

演習の成果

工夫点，苦心点，強調点，感想

Web開発の基礎を構成する技術として、HTML、CSS、JavaScript、そしてDOMがあります。これらの技術を駆使することで、魅力的でインタラクティブなウェブサイトを作成することが可能です。以下に、それぞれの技術についての工夫点、苦心点、強調点、そして使用した感想を述べます。まず、HTMLはウェブページの骨組みを形成するための言語です。見出しには ＜h1＞ から ＜h6＞タグを使い、ナビゲーションには ＜nav＞、メインコンテンツには ＜main＞を使用します。苦心点としては、HTML単体では動的なコンテンツを作れないことがあります。強調点はそのシンプルさにあり、構造をしっかり学ぶことで他の技術と組み合わせる際の理解が深まります。 CSSはHTMLにスタイルを適用するための言語です。webサイトを見やすいように、さまざまなデバイスに応じてスタイルを調整することが重要な工夫点です。フレックスボックスやグリッドレイアウトを使うことで、レイアウトの管理が容易になります。苦心点としては、ブラウザ間の互換性問題があります。異なるブラウザがCSSのプロパティを異なる解釈することがあるため、これに対応するのは難しいです。強調点は、CSS3の登場でアニメーションやトランジションなど、ビジュアル表現の幅が広がったことです。これにより、よりダイナミックで美しいデザインが可能になります。 JavaScriptはウェブページにインタラクティブな要素を追加するための言語です。工夫点として、色々なコードでwebページに動きをつけることができます。buttunをつけたりしました。苦心点は、非同期処理やコールバックの理解が難しい点です。動的型付けによるバグも発生しやすいです。 DOMは、HTMLやXML文書を操作するためのインターフェースです。効率的なDOM操作のためには、イベントデリゲーションを用いてイベントリスナーを管理し、パフォーマンスを向上させることが求められます。苦心点としては、複雑な文書構造を操作する際に特定の要素を選択して操作することが難しいことがあります。強調点は、プログラム可能な構造を提供することで、JavaScriptと組み合わせて動的なコンテンツを実現できる点です。 これらの技術を組み合わせることで、豊かでインタラクティブなウェブサイトを構築できます。HTML、CSS、JavaScript、DOMをマスターすることは、ウェブ開発者としての基礎スキルを確立し、新しい技術に対する柔軟性を持つことにつながります。技術の進化が早いこの分野において、これらの基本技術をしっかりと理解し、適応する能力を磨くことは、ウェブ開発の可能性を広げるために非常に大切だと考えます。
